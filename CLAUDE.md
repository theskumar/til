# CLAUDE.md — AI Assistant Guide for theskumar/til

## Project Overview

This is a personal **TIL (Things I Learned)** repository — a curated knowledge base of technical
learnings with automated weekly digests and an RSS feed. It is inspired by
[jakelazaroff/til](https://github.com/jakelazaroff/til/).

- **Content language:** Markdown
- **Automation language:** JavaScript (Node.js)
- **CI/CD:** GitHub Actions (`.github/workflows/build.yml`)
- **RSS:** Published at `https://saurabh-kumar.com/til/rss.xml`

---

## Repository Structure

```
til/
├── til.md                  # Primary notes file — all dated entries live here
├── extract-weekly.js       # Script: extracts dated notes → weekly/*.md
├── package.json            # NPM deps: marked (^17.0.3), he (^1.2.0)
├── rss.xml                 # Auto-generated RSS feed (do not edit manually)
├── README.md               # Auto-generated index (do not edit below the --- line)
├── .github/
│   └── workflows/
│       └── build.yml       # CI: generates weekly notes, README index, and RSS
├── weekly/                 # Auto-generated weekly summaries (YYYY-MM-DD.md)
│   └── 2025-12-20.md … 2026-02-22.md
├── docs/                   # Meta-documentation
├── fts/                    # Notes on full-text search
├── github/                 # Notes on GitHub
├── react/                  # Notes on React
└── vscode/                 # Notes on VS Code
```

### Category folders

Individual, polished TIL articles live in topic subfolders. Each file:
- Has a kebab-case filename (e.g., `pg_textsearch.md`)
- Starts with a single `# H1 Title` (used for README index and RSS titles)
- Contains a focused, self-contained write-up

## Content Conventions

See [README.md § Contributing](./README.md#contributing) for the contribution format and workflow.

Technical details the parser enforces:
- Date regex: `/^- (?<day>\d{1,2}) (?<month>[A-Za-z]{3}) (?<year>\d{4})/`
- Continuation lines must use exactly **2 spaces** — anything else terminates the note.
- `# Month YYYY` section headers in `til.md` are cosmetic; they are not parsed.
- Category detection is folder-based: `<category>/<slug>.md` → `## <category>` in README.
- RSS titles come from the first `# Heading` in each article file.

### README.md structure

Split by `---` into two sections:
- **Above `---`**: header + contributing guide — edit freely.
- **Below `---`**: index — **auto-generated by CI, do not edit manually**.

## Automation

### extract-weekly.js

Parses `til.md` for dated notes and writes one file per completed week to `weekly/`.

```bash
node extract-weekly.js         # Generate only missing weekly files
node extract-weekly.js --all   # Regenerate all weekly files
```

- Weeks end on **Sunday**; the current incomplete week is skipped.
- Output filename: `weekly/YYYY-MM-DD.md` (date = the ending Sunday).
- Notes within a week are sorted newest-first.

### CI/CD Pipeline (`.github/workflows/build.yml`)

Triggers:
- Push to `main`
- Weekly cron: every Sunday at 00:00 UTC
- Manual `workflow_dispatch`

Steps:
1. **Checkout** with full history (`fetch-depth: 0`)
2. **`node extract-weekly.js`** — generate `weekly/*.md`
3. **Regenerate README** — scans all `**/*.md` (excluding `README.md`, `til.md`, top-level files), groups by category, extracts `# Title`, rebuilds the index below `---`
4. **`npm install marked he`** — install RSS dependencies
5. **Generate `rss.xml`** — 10 most-recently-modified articles, Markdown → HTML via `marked`, HTML-escaped via `he`, RSS 2.0 with `content:encoded`
6. **Commit and push** as `TIL Bot` if any files changed

## Notes for AI Assistants

- **Adding content**: Always add quick notes to `til.md` using the `- DD Mon YYYY.` format. Standalone articles go in a category subfolder with a `# Title` header.
- **Do not regenerate** `README.md` index, `rss.xml`, or `weekly/` files manually — CI handles these.
- **Date format is strict**: `DD Mon YYYY` (e.g., `20 Feb 2026`). The `extract-weekly.js` parser uses the regex `/^- (?<day>\d{1,2}) (?<month>[A-Za-z]{3}) (?<year>\d{4})/`.
- **Continuation lines** in `til.md` must use exactly 2-space indentation; anything else terminates the note.
- **Category detection** is folder-based: a file at `<category>/<slug>.md` appears under `## <category>` in the README.
- **RSS titles** come from the first `# Heading` in each article file.
- The CI bot commits as `TIL Bot <actions@users.noreply.github.com>`.
